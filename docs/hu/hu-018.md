## 👤 Historia de Usuario

**Como** usuario del sistema  
**Quiero** gestionar los tipos de empleado (Vendedor, Cajero, Gerente, Administrador, Oficios, etc.)  
**Para** poder categorizar a los empleados según su rol dentro de la empresa

---

## 🧠 Descripción Funcional

El sistema debe permitir crear, editar, eliminar y listar los tipos de empleado. Cada tipo de empleado tendrá un nombre único y una descripción opcional. El sistema debe validar que no se puedan eliminar tipos de empleado que estén asociados a empleados existentes.

---

## ✅ Criterios de Aceptación

### Escenario 1: Crear tipo de empleado
- Dado que el usuario está en la pantalla de gestión de tipos de empleado
- Cuando hace clic en "Nuevo Tipo de Empleado" e ingresa un nombre válido y único
- Entonces el sistema crea el tipo de empleado y lo muestra en la lista

### Escenario 2: Editar tipo de empleado
- Dado que el usuario está en la lista de tipos de empleado
- Cuando selecciona un tipo de empleado y modifica su nombre
- Entonces el sistema actualiza el registro y muestra la confirmación

### Escenario 3: Eliminar tipo de empleado no utilizado
- Dado que el usuario está en la lista de tipos de empleado
- Cuando selecciona un tipo de empleado que NO está asociado a ningún empleado
- Entonces el sistema elimina el registro y muestra la confirmación

### Escenario 4: Eliminar tipo de empleado en uso
- Dado que el usuario está en la lista de tipos de empleado
- Cuando selecciona un tipo de empleado que ESTÁ asociado a uno o más empleados
- Entonces el sistema muestra un mensaje de error indicando que no se puede eliminar

### Escenario 5: Listar tipos de empleado
- Dado que el usuario accede a la gestión de tipos de empleado
- Cuando la pantalla carga
- Entonces el sistema muestra una lista paginada con todos los tipos de empleado de la empresa

### Escenario 6: Validar nombre único
- Dado que el usuario intenta crear o editar un tipo de empleado
- Cuando ingresa un nombre que ya existe en la empresa
- Entonces el sistema muestra un mensaje de error indicando que el nombre ya está en uso

---

## ❌ Casos de Error

- Si el nombre está vacío, mostrar error "El nombre es requerido"
- Si el nombre ya existe, mostrar error "Ya existe un tipo de empleado con este nombre"
- Si se intenta eliminar un tipo en uso, mostrar error "No se puede eliminar, existen empleados asociados"

---

## 🔐 Reglas de Negocio

- El nombre del tipo de empleado debe ser único por empresa
- Los tipos de empleado predefinidos inicial mente son: VENDEDOR, CAJERO, GERENTE, ADMINISTRADOR, OFICIOS
- El tipo de empleado es obligatorio al crear/editar un empleado (no puede ser nulo)

---

## 🎨 Consideraciones UI/UX

- Mostrar mensaje de confirmación antes de eliminar
- Resaltar campos requeridos con asterisco
- Mostrar spinner de carga durante operaciones
- Mostrar mensajes toast de éxito/error

---

## 📡 Requisitos Técnicos

- Endpoint GET /api/tipos-empleado - Listar todos
- Endpoint POST /api/tipos-empleado - Crear
- Endpoint PUT /api/tipos-empleado/{id} - Editar
- Endpoint DELETE /api/tipos-empleado/{id} - Eliminar
- Todos los endpoints filtran por empresa_id del JWT

---

## 📎 Dependencias

- Tabla: tipo_empleado (empresa_id, nombre, descripcion, activo, created_at, updated_at)
- Relación con empleados (uno a muchos)

---

## 🚫 Fuera de Alcance

- No incluye gestión de permisos por tipo de empleado
- No incluye reporte de empleados por tipo
