# HU-009: Gestión de Cuentas por Cobrar

## Descripción
Permite listar, filtrar, crear manualmente y obtener el detalle de las cuentas por cobrar de clientes. Una cuenta por cobrar representa una deuda pendiente de un cliente, ya sea originada por una venta o creada manualmente.

## Criterios de Aceptación

### Listar Cuentas por Cobrar
- [x] Endpoint: `POST /api/cuentas-cobrar/page`
- [x] Request: `PageableDto` con filtros (search, fechaDesde, fechaHasta, clienteId, estado)
- [x] Filtros disponibles:
  - `search`: búsqueda por número de cuenta, razón social o documento del cliente
  - `fechaDesde`: filtro por fecha de emisión inicial
  - `fechaHasta`: filtro por fecha de emisión final
  - `clienteId`: filtro por ID del cliente (tercero)
  - `estado`: filtro por estado (activa, pagada, vencida) - calculado dinámicamente
- [x] Response: `ApiResponse<PageImpl<CuentaCobrarTableDto>>`
- [x] Campos en tabla: id, numero_cuenta, cliente_nombre, cliente_documento, fecha_emision, fecha_vencimiento, total_deuda, total_abonado, saldo_pendiente, estado
- [x] Paginación con COUNT(*) OVER() AS total_rows
- [x] Orden: id DESC
- [x] **Nota**: Los totales se calculan dinámicamente desde abonos (HU-015)

### Crear Cuenta por Cobrar Manual
- [x] Endpoint: `POST /api/cuentas-cobrar`
- [x] Request: `CreateCuentaCobrarDto`
  - `clienteId` (Long, requerido): ID del tercero cliente
  - `totalDeuda` (BigDecimal, requerido): monto total de la deuda
  - `fechaEmision` (LocalDateTime, requerido): fecha de emisión
  - `fechaVencimiento` (LocalDateTime, opcional): fecha de vencimiento
  - `observaciones` (String, opcional): notas adicionales
- [x] Validaciones:
  - El tercero debe existir y tener es_cliente = true
  - totalDeuda debe ser mayor a 0
  - Si existe ventaId, debe ser una venta válida de la empresa
- [x] Response: `ApiResponse<CuentaCobrarDto>` con status 201
- [x] El número de cuenta se genera automáticamente: CC-YYYYMMDD-XXXX

### Obtener Detalle de Cuenta por Cobrar
- [x] Endpoint: `GET /api/cuentas-cobrar/{id}`
- [x] Response: `ApiResponse<CuentaCobrarDto>`
- [x] Incluye lista de abonos realizados
- [x] Status 404 si no existe o no pertenece a la empresa

### Actualizar Cuenta por Cobrar
- [x] Endpoint: `PUT /api/cuentas-cobrar/{id}`
- [x] Request: `UpdateCuentaCobrarDto` (todos los campos opcionales)
- [x] Solo permite modificar: fechaVencimiento, observaciones
- [x] No permite modificar montos una vez hay abonos registrados

## Seguridad
- [x] Requiere JWT válido
- [ ] Filtra por empresa_id del token
- [ ] Roles permitidos: ADMIN, CAJERO, VENDEDOR

## Database Modifications

### Tabla: cuentas_cobrar
```sql
CREATE TABLE IF NOT EXISTS cuentas_cobrar (
    id BIGSERIAL PRIMARY KEY,
    empresa_id BIGINT NOT NULL REFERENCES empresa(id),
    venta_id BIGINT REFERENCES venta(id),
    tercero_id BIGINT NOT NULL REFERENCES tercero(id),
    numero_cuenta VARCHAR(20) NOT NULL UNIQUE,
    fecha_emision TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    fecha_vencimiento TIMESTAMP,
    total_deuda DECIMAL(15,2) NOT NULL,
    total_abonado DECIMAL(15,2) NOT NULL DEFAULT 0,
    saldo_pendiente DECIMAL(15,2) NOT NULL,
    estado VARCHAR(20) NOT NULL DEFAULT 'activa' CHECK (estado IN ('activa', 'pagada', 'vencida')),
    observaciones TEXT,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP,
    deleted_at TIMESTAMP
);

-- Índices
CREATE INDEX idx_cuentas_cobrar_empresa ON cuentas_cobrar(empresa_id);
CREATE INDEX idx_cuentas_cobrar_tercero ON cuentas_cobrar(tercero_id);
CREATE INDEX idx_cuentas_cobrar_estado ON cuentas_cobrar(estado);
CREATE INDEX idx_cuentas_cobrar_fecha_emision ON cuentas_cobrar(fecha_emision);
CREATE INDEX idx_cuentas_cobrar_numero ON cuentas_cobrar(numero_cuenta);
```

### Tabla: abonos_cobrar
```sql
CREATE TABLE IF NOT EXISTS abonos_cobrar (
    id BIGSERIAL PRIMARY KEY,
    cuenta_cobrar_id BIGINT NOT NULL REFERENCES cuentas_cobrar(id),
    usuario_id BIGINT NOT NULL REFERENCES usuario(id),
    monto DECIMAL(15,2) NOT NULL,
    metodo_pago VARCHAR(30) NOT NULL,
    referencia VARCHAR(100),
    fecha_pago TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMP
);

-- Índices
CREATE INDEX idx_abonos_cobrar_cuenta ON abonos_cobrar(cuenta_cobrar_id);
CREATE INDEX idx_abonos_cobrar_fecha ON abonos_cobrar(fecha_pago);
```
