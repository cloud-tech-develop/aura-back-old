# HU-011: Abonos a Cuentas por Cobrar

## Descripción
Permite registrar abonos (pagos) a cuentas por cobrar. Un abono representa una entrada de caja que reduce el saldo pendiente de una cuenta por cobrar.

## Criterios de Aceptación

### Registrar Abono a Cuenta por Cobrar
- [x] Endpoint: `POST /api/cuentas-cobrar/{id}/abonos`
- [x] Request: `AbonoCobrarDto`
  - `monto` (BigDecimal, requerido): monto del abono
  - `metodoPago` (String, requerido): método de pago (EFECTIVO, TRANSFERENCIA, TARJETA, CHEQUE, OTRO)
  - `referencia` (String, opcional): número de referencia del pago
  - `fechaPago` (LocalDateTime, opcional): fecha del pago (default: ahora)
- [x] Validaciones:
  - La cuenta por cobrar debe existir y pertenecer a la empresa
  - El monto debe ser mayor a 0
  - El monto no puede ser mayor al saldo pendiente
  - La cuenta no debe estar en estado 'pagada'
- [x] Response: `ApiResponse<AbonoCobrarDto>` con status 201
- [x] Transaccional: registra el abono y actualiza el saldo de la cuenta

### Actualizar saldo de cuenta automáticamente
- [x] El abono se registra con el usuario_id del JWT
- **Nota**: Los totales se calculan dinámicamente desde abonos (ver HU-015)

### Listar Abonos de una Cuenta
- [x] Endpoint: `GET /api/cuentas-cobrar/{id}/abonos`
- [x] Response: `ApiResponse<List<AbonoCobrarDto>>`
- [x] Incluye: id, monto, metodoPago, referencia, fechaPago, usuarioNombre, createdAt
- [x] Orden: fecha_pago DESC

### Obtener Abono por ID
- [x] Endpoint: `GET /api/cuentas-cobrar/{cuentaId}/abonos/{abonoId}`
- [x] Response: `ApiResponse<AbonoCobrarDto>`

### Eliminar Abono (Anulación)
- [x] Endpoint: `DELETE /api/cuentas-cobrar/{cuentaId}/abonos/{abonoId}`
- [x] Solo permite anulación si el abono es del día actual
- [x] Validaciones:
  - No permite si la cuenta ya está marcada como pagada
  - No permite si al eliminar el abono el saldo queda negativo
- [x] Transaccional: reversa el abono y recalcula saldo
- [x] Response: `ApiResponse<Void>` con status 200

## Seguridad
- [x] Requiere JWT válido
- [x] Filtra por empresa_id del token
- [x] Roles permitidos: ADMIN, CAJERO

## Cálculos Automáticos (ver HU-015)
Los totales se calculan dinámicamente desde la tabla de abonos:
```
total_abonado = COALESCE(SUM(monto) FROM abonos_cobrar WHERE cuenta_cobrar_id = ? AND deleted_at IS NULL)
saldo_pendiente = total_deuda - total_abonado

estado = 
  IF saldo_pendiente <= 0 THEN 'pagada'
  ELSE IF fecha_vencimiento < NOW() THEN 'vencida'
  ELSE 'activa'
```
