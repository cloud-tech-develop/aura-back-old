# HU-015: Validación de Totales de Cuentas desde Abonos

## Descripción
Los campos `total_abonado` y `saldo_pendiente` en las tablas de cuentas por cobrar y pagar deben calcularse dinámicamente desde los abonos registrados, no desde valores almacenados manualmente. Esto garantiza la integridad de los datos.

## Problema Actual
Actualmente, al registrar un abono, el servicio actualiza manualmente los campos:
- `total_abonado` = `total_abonado + monto_abono`
- `saldo_pendiente` = `saldo_pendiente - monto_abono`

Esto puede causar inconsistencias si:
1. Se modifican abonos directamente en la BD
2. Hay errores en la lógica de actualización
3. Se eliminan abonos sin actualizar la cuenta

## Solución Propuesta
Los totales deben calcularse dinámicamente desde la tabla de abonos:

```sql
-- Cálculo correcto:
total_abonado = COALESCE((SELECT SUM(monto) FROM abonos_pagar WHERE cuenta_pagar_id = cp.id AND deleted_at IS NULL), 0)
saldo_pendiente = cp.total_deuda - total_abonado
```

## Criterios de Aceptación

### En QueryRepository (lectura)
- [x] `total_abonado` se calcula: `SUM(monto)` de abonos relacionados
- [x] `saldo_pendiente` se calcula: `total_deuda - total_abonado`
- [x] El `estado` se calcula automáticamente:
  - Si `saldo_pendiente <= 0` → "pagada"
  - Si `saldo_pendiente > 0` Y `fecha_vencimiento < NOW()` → "vencida"
  - Si `saldo_pendiente > 0` Y `fecha_vencimiento >= NOW()` → "activa"

### En Servicio (escritura)
- [x] Al crear cuenta: inicializar `total_abonado = 0`, `saldo_pendiente = total_deuda` (como referencia inicial)
- [x] Al registrar abono: NO actualizar manualmente los campos de la cuenta
- [x] Al eliminar abono: NO actualizar manualmente los campos

### Métodos a modificar
1. `listar()` - calcular totales desde abonos
2. `listarConFiltros()` - calcular totales desde abonos
3. `obtenerPorId()` - calcular totales desde abonos
4. `obtenerResumen()` - calcular totales desde abonos
5. `obtenerVencidas()` - calcular totales desde abonos

### Aplicar a
- [x] Cuentas por Pagar (cuentas_pagar)
- [x] Cuentas por Cobrar (cuentas_cobrar)

## Cambios en SQL

### Para cuentas_pagar:
```sql
SELECT
    cp.id,
    cp.numero_cuenta,
    cp.total_deuda,
    COALESCE((SELECT SUM(monto) FROM abonos_pagar WHERE cuenta_pagar_id = cp.id AND deleted_at IS NULL), 0) AS total_abonado,
    cp.total_deuda - COALESCE((SELECT SUM(monto) FROM abonos_pagar WHERE cuenta_pagar_id = cp.id AND deleted_at IS NULL), 0) AS saldo_pendiente,
    CASE 
        WHEN cp.total_deuda - COALESCE((SELECT SUM(monto) FROM abonos_pagar WHERE cuenta_pagar_id = cp.id AND deleted_at IS NULL), 0) <= 0 THEN 'pagada'
        WHEN cp.fecha_vencimiento < NOW() THEN 'vencida'
        ELSE 'activa'
    END AS estado
FROM cuentas_pagar cp
...
```

### Para cuentas_cobrar:
```sql
SELECT
    cc.id,
    cc.numero_cuenta,
    cc.total_deuda,
    COALESCE((SELECT SUM(monto) FROM abonos_cobrar WHERE cuenta_cobrar_id = cc.id AND deleted_at IS NULL), 0) AS total_abonado,
    cc.total_deuda - COALESCE((SELECT SUM(monto) FROM abonos_cobrar WHERE cuenta_cobrar_id = cc.id AND deleted_at IS NULL), 0) AS saldo_pendiente,

        WHEN cc.total_deuda - COALESCE((SELECT SUM(monto) FROM abonos_cobrar WHERE cuenta_cobrar_id = cc.id AND deleted_at IS NULL), 0) <= 0 THEN 'pagada'

```
