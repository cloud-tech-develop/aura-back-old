## 👤 Historia de Usuario

**Como** usuario del sistema  
**Quiero** crear y gestionar rutas de visita  
**Para** organizar la planificación de visitas de los vendedores a los locales

---

## 🧠 Descripción Funcional

El sistema debe permitir crear rutas que agrupen locales específicos para ser visitados en un orden determinado. Cada ruta tiene un nombre, descripción, vendedor asignado y una lista de locales ordenados.

---

## ✅ Criterios de Aceptación

### Escenario 1: Crear ruta
- Dado que el usuario está en la pantalla de gestión de rutas
- Cuando hace clic en "Nueva Ruta" e ingresa nombre, descripción, selecciona vendedor y agrega locales
-Entonces el sistema crea la ruta con los locales asociados

### Escenario 2: Editar ruta
- Dado que el usuario está en la lista de rutas
- Cuando selecciona una ruta y modifica su información o el orden de locales
-Entonces el sistema actualiza el registro

### Escenario 3: Eliminar ruta
- Dado que el usuario está en la lista de rutas
- Cuando selecciona una ruta y confirma la eliminación
-Entonces el sistema elimina el registro

### Escenario 4: Listar rutas
- Dado que el usuario accede a la gestión de rutas
- Cuando la pantalla carga
-Entonces el sistema muestra una lista paginada de rutas de la empresa

### Escenario 5: Ver detalles de ruta
-Dado que el usuario está en la lista de rutas
- Cuando hace clic en ver detalles de una ruta
-Entonces el sistema muestra la información completa incluyendo el mapa con los locales ordenados

### Escenario 6: Reordenar locales en ruta
- Dado que el usuario está editando una ruta
- Cuando arrastra los locales para cambiar su orden
-Entonces el sistema actualiza el orden de visita

### Escenario 7: Filtrar rutas por vendedor
- Dado que el usuario está en la lista de rutas
- Cuando selecciona un vendedor del filtro
-Entonces el sistema muestra solo las rutas de ese vendedor

### Escenario 8: Precargar ruta anterior
- Dado que el usuario está en el formulario de crear ruta
- Cuando hace clic en "Cargar Ruta Anterior" y selecciona una ruta existente
-Entonces el sistema carga los datos de esa ruta (nombre, descripción, vendedor, locales) permitiendo modificarlos

### Escenario 9: Modificar ruta precargada
- Dado que el usuario tiene una ruta precargada
- Cuando agrega o elimina locales de la lista
-Entonces el sistema actualiza la lista de locales de la ruta

### Escenario 10: Nueva ruta desde cero
- Dado que el usuario está en el formulario de crear ruta
- Cuando no selecciona ninguna ruta anterior
-Entonces el sistema muestra el formulario vacío para crear desde cero

---

## ❌ Casos de Error

- Si el nombre está vacío, mostrar error "El nombre es requerido"
- Si no hay locales asociados, mostrar error "Debe agregar al menos un local"
- Si se intenta eliminar una ruta con visitas asociadas, mostrar error

---

## 🔐 Reglas de Negocio

- Una ruta pertenece a un vendedor específico
- Los locales en una ruta tienen un orden específico de visita
- Las rutas se filtran por empresa_id del JWT
- La precarga de ruta anterior permite copiar todos los datos de una ruta existente
- Al precargar se crea una copia, no se modifica la ruta original
- Se puede agregar o quitar locales de la ruta precargada

---

## 🎨 Consideraciones UI/UX

- Interfaz de drag-and-drop para reordenar locales
- Mapa mostrando la ruta completa con los pines de los locales
- Indicador visual del orden de visita
- Mostrar tiempo estimado entre locales

---

## 📡 Requisitos Técnicos

- Endpoints REST para CRUD de rutas
- La ruta contiene lista de locales con campo "orden" (integer)
- GET /api/rutas/{id}/locales para obtener locales de una ruta
