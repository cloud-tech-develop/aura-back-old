# HU-006 - Validación de Cliente Obligatorio para Ventas a Crédito

---

## 👤 Historia de Usuario

**Como** sistema de punto de venta  
**Quiero** validar que toda venta a crédito tenga un cliente asociado  
**Para** garantizar la trazabilidad de las cuentas por cobrar y facilitar la gestión de cobros

---

## 🧠 Descripción Funcional

El sistema debe validar que toda venta que utilize el método de pago "crédito" tenga un cliente (tercero) asociado. Esta validación debe realizarse al momento de crear la venta y debe ser un requisito obligatorio para poder completar la transacción.

---

## ✅ Criterios de Aceptación

### Escenario 1: Venta a crédito con cliente válido
- Dado que se intenta crear una venta con método de pago "crédito"
- Y se proporciona un cliente_id válido
- Cuando se procesa la venta
- Entonces:
  - La venta debe crearse exitosamente
  - El cliente debe estar asociado a la venta
  - El campo `pago_parcial` debe ser true
  - El campo `saldo_pendiente` debe ser igual al total de la venta

### Escenario 2: Venta a crédito sin cliente
- Dado que se intenta crear una venta con método de pago "crédito"
- Y NO se proporciona cliente_id
- Cuando se procesa la venta
- Entonces:
  - Debe retornar error 400
  - El mensaje debe indicar "Las ventas a crédito requieren un cliente asociado"

### Escenario 3: Venta a crédito con cliente inexistente
- Dado que se intenta crear una venta con método de pago "crédito"
- Y se proporciona un cliente_id que no existe
- Cuando se procesa la venta
- Entonces:
  - Debe retornar error 404
  - El mensaje debe indicar "Cliente no encontrado"

### Escenario 4: Venta en efectivo sin cliente
- Dado que se intenta crear una venta con método de pago "efectivo"
- Y NO se proporciona cliente_id
- Cuando se procesa la venta
- Entonces:
  - La venta debe crearse exitosamente (cliente opcional)

### Escenario 5: Venta con múltiples pagos incluyendo crédito
- Dado que se intenta crear una venta con pagos: [$50.000 efectivo + $50.000 crédito]
- Y se proporciona un cliente_id válido
- Cuando se procesa la venta
- Entonces:
  - La venta debe crearse exitosamente
  - Debe validarse que existe cliente para la porción de crédito

---

## 🔐 Reglas de Negocio

- El método de pago "crédito" siempre requiere cliente
- Si uno de los métodos de pago es "crédito", se requiere cliente
- El cliente debe existir y pertenecer a la empresa
- Para ventas 100% en efectivo, transferencia o tarjeta, el cliente es opcional

---

## 📡 Lógica de Validación

```java
public boolean requiereCliente(List<CreateVentaPagoDto> pagos) {
    return pagos.stream()
        .anyMatch(p -> "credito".equalsIgnoreCase(p.getMetodoPago()));
}
```
