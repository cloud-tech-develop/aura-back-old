# HU-010: Gestión de Cuentas por Pagar

## Descripción
Permite listar, filtrar, crear manualmente y obtener el detalle de las cuentas por pagar a proveedores. Una cuenta por pagar representa una deuda pendiente con un proveedor, ya sea originada por una compra o creada manualmente.

## Criterios de Aceptación

### Listar Cuentas por Pagar
- [x] Endpoint: `POST /api/cuentas-pagar/page`
- [x] Request: `PageableDto` con filtros (search, fechaDesde, fechaHasta, proveedorId, estado)
- [x] Filtros disponibles:
  - `search`: búsqueda por número de cuenta, razón social o documento del proveedor
  - `fechaDesde`: filtro por fecha de emisión inicial
  - `fechaHasta`: filtro por fecha de emisión final
  - `proveedorId`: filtro por ID del proveedor (tercero)
  - `estado`: filtro por estado (activa, pagada, vencida) - calculado dinámicamente
- [x] Response: `ApiResponse<PageImpl<CuentaPagarTableDto>>`
- [x] Campos en tabla: id, numero_cuenta, proveedor_nombre, proveedor_documento, fecha_emision, fecha_vencimiento, total_deuda, total_abonado, saldo_pendiente, estado
- [x] Paginación con COUNT(*) OVER() AS total_rows
- [x] Orden: id DESC
- [x] **Nota**: Los totales se calculan dinámicamente desde abonos (HU-015)

### Crear Cuenta por Pagar Manual
- [x] Endpoint: `POST /api/cuentas-pagar`
- [x] Request: `CreateCuentaPagarDto`
  - `proveedorId` (Long, requerido): ID del tercero proveedor
  - `totalDeuda` (BigDecimal, requerido): monto total de la deuda
  - `fechaEmision` (LocalDateTime, requerido): fecha de emisión
  - `fechaVencimiento` (LocalDateTime, opcional): fecha de vencimiento
  - `observaciones` (String, opcional): notas adicionales
- [x] Validaciones:
  - El tercero debe existir y tener es_proveedor = true
  - totalDeuda debe ser mayor a 0
  - Si existe compraId, debe ser una compra válida de la empresa
- [x] Response: `ApiResponse<CuentaPagarDto>` con status 201
- [x] El número de cuenta se genera automáticamente: CP-YYYYMMDD-XXXX

### Obtener Detalle de Cuenta por Pagar
- [x] Endpoint: `GET /api/cuentas-pagar/{id}`
- [x] Response: `ApiResponse<CuentaPagarDto>`
- [x] Incluye lista de abonos realizados
- [x] Status 404 si no existe o no pertenece a la empresa

### Actualizar Cuenta por Pagar
- [x] Endpoint: `PUT /api/cuentas-pagar/{id}`
- [x] Request: `UpdateCuentaPagarDto` (todos los campos opcionales)
- [x] Solo permite modificar: fechaVencimiento, observaciones
- [x] No permite modificar montos una vez hay abonos registrados

## Seguridad
- [x] Requiere JWT válido
- [ ] Filtra por empresa_id del token
- [ ] Roles permitidos: ADMIN, CONTADOR, COMPRAS

## Database Modifications

### Tabla: cuentas_pagar
```sql
CREATE TABLE IF NOT EXISTS cuentas_pagar (
    id BIGSERIAL PRIMARY KEY,
    empresa_id BIGINT NOT NULL REFERENCES empresa(id),
    compra_id BIGINT REFERENCES compra(id),
    tercero_id BIGINT NOT NULL REFERENCES tercero(id),
    numero_cuenta VARCHAR(20) NOT NULL UNIQUE,
    fecha_emision TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    fecha_vencimiento TIMESTAMP,
    total_deuda DECIMAL(15,2) NOT NULL,
    total_abonado DECIMAL(15,2) NOT NULL DEFAULT 0,
    saldo_pendiente DECIMAL(15,2) NOT NULL,
    estado VARCHAR(20) NOT NULL DEFAULT 'activa' CHECK (estado IN ('activa', 'pagada', 'vencida')),
    observaciones TEXT,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP,
    deleted_at TIMESTAMP
);

-- Índices
CREATE INDEX idx_cuentas_pagar_empresa ON cuentas_pagar(empresa_id);
CREATE INDEX idx_cuentas_pagar_tercero ON cuentas_pagar(tercero_id);
CREATE INDEX idx_cuentas_pagar_estado ON cuentas_pagar(estado);
CREATE INDEX idx_cuentas_pagar_fecha_emision ON cuentas_pagar(fecha_emision);
CREATE INDEX idx_cuentas_pagar_numero ON cuentas_pagar(numero_cuenta);
```

### Tabla: abonos_pagar
```sql
CREATE TABLE IF NOT EXISTS abonos_pagar (
    id BIGSERIAL PRIMARY KEY,
    cuenta_pagar_id BIGINT NOT NULL REFERENCES cuentas_pagar(id),
    usuario_id BIGINT NOT NULL REFERENCES usuario(id),
    monto DECIMAL(15,2) NOT NULL,
    metodo_pago VARCHAR(30) NOT NULL,
    referencia VARCHAR(100),
    banco VARCHAR(100),
    fecha_pago TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMP
);

-- Índices
CREATE INDEX idx_abonos_pagar_cuenta ON abonos_pagar(cuenta_pagar_id);
CREATE INDEX idx_abonos_pagar_fecha ON abonos_pagar(fecha_pago);
```
