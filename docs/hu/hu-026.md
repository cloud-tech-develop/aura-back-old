## 👤 Historia de Usuario

**Como** usuario del sistema  
**Quiero** listar todos los locales de la empresa  
**Para** tener una visión global de todos los puntos de venta y su distribución

---

## 🧠 Descripción Funcional

El sistema debe mostrar una lista completa de todos los locales registrados en la empresa, con la capacidad de filtrar y buscar. Esta vista permite al usuario ver la cartera completa de clientes de la empresa.

---

## ✅ Criterios de Aceptación

### Escenario 1: Listar todos los locales
- Dado que el usuario accede a la pantalla de locales
- Cuando la pantalla carga
-Entonces el sistema muestra una lista paginada de todos los locales de la empresa

### Escenario 2: Buscar local por nombre
- Dado que el usuario está en la lista de locales
- Cuando ingresa un término de búsqueda en el campo de búsqueda
-Entonces el sistema filtra los locales que coinciden con el término

### Escenario 3: Filtrar por vendedor actual
- Dado que el usuario está en la lista de locales
- Cuando selecciona un vendedor del filtro
-Entonces el sistema muestra solo los locales de ese vendedor

### Escenario 4: Filtrar por vendedor anterior
- Dado que el usuario está en la lista de locales
- Cuando selecciona un vendedor anterior del filtro
-Entonces el sistema muestra solo los locales que tuvieron ese vendedor como anterior

### Escenario 5: Ordenar por columnas
- Dado que el usuario está en la lista de locales
- Cuando hace clic en el encabezado de una columna
-Entonces el sistema ordena la lista por esa columna (asc/desc)

### Escenario 6: Ver detalles del local
- Dado que el usuario está en la lista de locales
- Cuando hace clic en un local
-Entonces el sistema muestra los detalles completos del local

### Escenario 7: Ver distribución en mapa
- Dado que el usuario está en la lista de locales
- Cuando hace clic en "Ver en Mapa"
-Entonces el sistema muestra un mapa con todos los locales con sus pines

---

## ❌ Casos de Error

- Si no hay locales registrados, mostrar mensaje "No hay locales registrados"
- Si hay un error de base de datos, mostrar mensaje de error genérico

---

## 🔐 Reglas de Negocio

- Se filtran por empresa_id del JWT
- Incluir paginación (tamaño configurable)
- Soportar ordenamiento por nombre, dirección, vendedor actual

---

## 🎨 Consideraciones UI/UX

- Campo de búsqueda con debounce
- Filtros combinados (vendedor, estado)
- Paginación con selector de tamaño de página
- Botón para ver mapa con todos los locales
- Indicador de cantidad de locales totales

---

## 📡 Requisitos Técnicos

- Endpoint GET /api/locales - Lista paginada con filtros
- Parámetros: page, size, sort, search, vendedorActualId, vendedorAnteriorId

- HU-022: Asignación de Locales a Vendedores
