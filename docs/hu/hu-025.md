## 👤 Historia de Usuario

**Como** usuario del sistema  
**Quiero** listar los locales asignados a un vendedor específico  
**Para** visualizar los clientes que atiende cada vendedor

---

## 🧠 Descripción Funcional

El sistema debe mostrar una lista de los locales que tienen asignado como vendedor actual a un vendedor específico. Esta funcionalidad permite ver la cartera de clientes de cada vendedor.

---

## ✅ Criterios de Aceptación

### Escenario 1: Listar locales de un vendedor
- Dado que el usuario selecciona un vendedor del dropdown
- Cuando hace clic en "Buscar" o la lista carga automáticamente
-Entonces el sistema muestra los locales asignados a ese vendedor

### Escenario 2: Ver información del local
- Dado que el usuario está en la lista de locales por vendedor
- Cuando hace clic en un local
-Entonces el sistema muestra los detalles completos del local

### Escenario 3: Ver vendedor anterior
- Dado que el usuario está viendo los detalles de un local
- Cuando visualiza la información
-Entonces el sistema muestra el nombre del vendedor anterior (si existe)

### Escenario 4: Sin locales asignados
- Dado que el usuario selecciona un vendedor que no tiene locales
- Cuando carga la lista
-Entonces el sistema muestra mensaje "Este vendedor no tiene locales asignados"

### Escenario 5: Dropdown de vendedores
- Dado que el usuario accede a la pantalla de locales por vendedor
- Cuando el dropdown de vendedores carga
-Entonces el sistema muestra solo los empleados con tipo "Vendedor" activos

---

## ❌ Casos de Error

- Si el vendedor no existe, mostrar error
- Si hay un error de base de datos, mostrar mensaje de error genérico

---

## 🔐 Reglas de Negocio

- Solo mostrar locales donde vendedor_actual_id sea igual al vendedor seleccionado
- Filtrar por empresa_id del JWT
- Incluir paginación

---

## 🎨 Consideraciones UI/UX

- Dropdown con filtro para buscar vendedores
- Tabla con información relevante (nombre, dirección, última visita)
- Indicador visual de locales sin visita reciente
- Link para ir a los detalles del local

---

## 📡 Requisitos Técnicos

- Endpoint GET /api/locales?vendedorActualId={id} - Filtrar por vendedor actual
- El dropdown usa GET /api/empleados?tipo=VENDEDOR
