# HU-019 - Asociación de Tipo de Empleado a Empleados

- Epic: EP-005
- Prioridad: Alta

- Porcentaje: 100%

- Fecha: 2026-03-27
- Frontend: [Ver docs](./../hu-frontend/hu-019-frontend.md)

---

## 👤 Historia de Usuario

**Como** usuario del sistema  
**Quiero** seleccionar un tipo de empleado al crear o editar un empleado  
**Para** categorizar correctamente al empleado según su rol dentro de la empresa

---

## 🧠 Descripción Funcional

Al crear o editar un empleado, el sistema debe mostrar un dropdown/listbox con los tipos de empleado disponibles (Vendedor, Cajero, Gerente, Administrador, Oficios, etc.) permitiendo seleccionar uno. El tipo de empleado es un campo obligatorio.

---

## ✅ Criterios de Aceptación

### Escenario 1: Crear empleado con tipo
- Dado que el usuario está en el formulario de creación de empleado
- Cuando selecciona un tipo de empleado del dropdown y completa los demás campos obligatorios
- Entonces el sistema crea el empleado con el tipo asociado

### Escenario 2: Editar tipo de empleado
- Dado que el usuario está en el formulario de edición de empleado
- Cuando cambia el tipo de empleado seleccionado
- Entonces el sistema actualiza el tipo de empleado del registro

### Escenario 3: Listar tipos en dropdown
- Dado que el usuario está en el formulario de crear/editar empleado
- Cuando el dropdown de tipo de empleado se despliega
- Entonces el sistema muestra todos los tipos de empleado activos de la empresa

### Escenario 4: Filtrar tipos por empresa
- Dado que el usuario está en el formulario de empleado
- Cuando carga el dropdown de tipos
-Entonces el sistema solo muestra los tipos de empleado de la empresa del usuario logueado

### Escenario 5: Campo obligatorio
- Dado que el usuario intenta guardar un empleado sin seleccionar tipo
- Cuando envía el formulario
- Entonces el sistema muestra error "El tipo de empleado es requerido"

---

## ❌ Casos de Error

- Si no hay tipos de empleado disponibles, mostrar mensaje "No hay tipos de empleado configurados"
- Si el tipo seleccionado no existe, mostrar error

---

## 🔐 Reglas de Negocio

- El tipo de empleado es obligatorio (not null)
- Solo se muestran tipos de empleado activos
- Se filtran por empresa_id del JWT

---

## 🎨 Consideraciones UI/UX

- El dropdown debe permitir búsqueda/filtrado (p.e. PrimeNG Dropdown con filter)
- Mostrar el nombre del tipo de empleado en el dropdown
- Mantener el valor seleccionado al mostrar errores de validación
- Deshabilitar la opción de crear nuevo tipo desde el dropdown (solo mostrar link a gestión de tipos)

---

## 📡 Requisitos Técnicos

- El endpoint GET /api/empleados/{id} debe incluir el tipo de empleado
- El endpoint POST /api/empleados debe aceptar tipo_empleado_id
- El endpoint PUT /api/empleados/{id} debe aceptar tipo_empleado_id

---

## 📎 Dependencias

- HU-018: Gestión de Tipos de Empleado (debe estar implementada para listar los tipos)
- Endpoint GET /api/tipos-empleado (de HU-018)

---

## 🚫 Fuera de Alcance

- No incluye creación de empleado desde el tipo de empleado
- No incluye filtrado de empleados por tipo en la lista principal
