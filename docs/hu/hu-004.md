## 👤 Historia de Usuario

**Como** sistema de punto de venta  
**Quiero** registrar pagos parciales en una venta y gestionar el saldo pendiente  
**Para** permitir a los clientes pagar parcialmente una compra y generar notas contables automáticas cuando el pago se complete

---

## 🧠 Descripción Funcional

El sistema debe permitir registrar pagos parciales en una venta. Cuando el total de pagos registrados sea menor al total de la venta, se considera una venta con pago parcial. El sistema debe calcular y mantener el saldo pendiente de pago, y cuando se complete el pago total, debe generar automáticamente las notas contables correspondientes (nota débito por el saldo剩余, nota crédito si hay devolución).

---

## ✅ Criterios de Aceptación

### Escenario 1: Registro de pago parcial en venta
- Dado que existe una venta con total de $100.000
- Cuando se registra un pago de $50.000
- Entonces:
  - El campo `pago_parcial` debe ser `true`
  - El campo `saldo_pendiente` debe ser $50.000
  - El campo `estado_venta` debe ser "PAGO_PARCIAL"

### Escenario 2: Venta sin pago parcial
- Dado que existe una venta con total de $100.000
- Cuando se registra un pago de $100.000
- Entonces:
  - El campo `pago_parcial` debe ser `false`
  - El campo `saldo_pendiente` debe ser $0
  - El campo `estado_venta` debe ser "COMPLETADA"

### Escenario 3: Registro de pagos parciales desde venta
- Dado que se genera una factura desde una venta con pago parcial
- Cuando se generan los recibos de pago
- Entonces:
  - Se debe crear un ReciboPago por cada pago registrado en VentaPago
  - La descripción del ReciboPago debe incluir: "Pago parcial - Saldo pendiente: $X" o información relevante de la venta

### Escenario 4: Completar pago parcial
- Dado que existe una venta con pago parcial de $50.000 restantes
- Cuando se registra un nuevo pago que completa el total
- Entonces:
  - El campo `saldo_pendiente` debe ser $0
  - El campo `estado_venta` debe cambiar a "COMPLETADA"
  - Debe generarse una NotaContable de tipo DÉBITO por el monto del último pago

### Escenario 5: Suma de pagos mayor al total
- Dado que existe una venta con total de $100.000
- Cuando se registra un pago de $120.000
- Entonces:
  - Debe calcularse el vuelto ($20.000)
  - Debe crearse una NotaContable de tipo CRÉDITO por el monto del vuelto

## 📡 Requisitos Técnicos

### Entidades

**VentaEntity:**
- Agregar `pagoParcial` (Boolean, default false)
- Agregar `saldoPendiente` (BigDecimal, default 0)

**ReciboPagoEntity:**
- La descripción debe incluir información de la venta y saldo pendiente si aplica

### Lógica de Negocio

1. Al registrar un pago:
   - Calcular: `pago_parcial = (suma_pagos < total_venta)`
   - Calcular: `saldo_pendiente = total_venta - suma_pagos`
   - Actualizar: `estado_venta` según corresponda

2. Al completar el pago (saldo_pendiente = 0):
   - Generar NotaContable DÉBITO por el último pago
   - Si suma_pagos > total_venta, generar NotaContable CRÉDITO por el sobrante
