## 👤 Historia de Usuario

**Como** sistema de compras  
**Quiero** registrar notas contables al pagarse una compra para controlar las salidas de dinero de la caja  
**Para** mantener trazabilidad de los pagos realizados a proveedores y el flujo de caja de la empresa

---

## 🧠 Descripción Funcional

Al completarse una compra en el sistema, el backend debe generar automáticamente notas contables de tipo DÉBITO para registrar la salida de dinero de la caja. Si la compra tiene un pago parcial (monto pagado < total compra), se genera una nota por el monto pagado. Si está pagada completamente, se genera una nota por el total de la factura de compra.

Las notas contables en compras representan egresos/salidas de dinero, a diferencia de las notas en ventas que pueden ser crédito o débito.

---

## ✅ Criterios de Aceptación

### Escenario 1: Compra pagada completamente - Nota por total
- Dado que se registra una compra con valor total de $500.000
- Y se registra un pago de $500.000 (pagado completamente)
- Cuando se completa la compra
- Entonces:
  - Se debe crear una NotaContable con tipo DÉBITO
  - El valor debe ser $500.000 (total de la factura de compra)
  - La nota debe indicar "Pago total Factura Compra #X"

### Escenario 2: Compra con pago parcial - Nota por monto pagado
- Dado que se registra una compra con valor total de $500.000
- Y se registra un pago de $200.000 (pago parcial)
- Cuando se completa la compra
- Entonces:
  - Se debe crear una NotaContable con tipo DÉBITO
  - El valor debe ser $200.000 (monto pagado)
  - La nota debe incluir: "Pago parcial Fact: {num_factura} - Usuario: {nombre_usuario}"

### Escenario 3: Compra con múltiples pagos
- Dado que se registra una compra con pagos: $200.000 + $300.000
- Cuando se completa la compra
- Entonces:
  - Debe crearse una NotaContable por cada pago realizado
  - Cada nota debe incluir la descripción del pago correspondiente

### Escenario 4: Compra a crédito (sin pago inicial)
- Dado que se registra una compra con método de pago "crédito"
- Y NO se registra ningún pago
- Cuando se completa la compra
- Entonces:
  - NO se debe crear ninguna nota contable en este momento
  - La nota se generará cuando se registre el pago

---

## 🔐 Reglas de Negocio

- Las notas contables en compras siempre son de tipo DÉBITO (salida de dinero)
- Si `monto_pagado < total_compra`, es un pago parcial
- El texto de nota para pago parcial debe seguir formato: "Pago parcial Fact: {prefijo}{consecutivo} - Usuario: {nombre_usuario}"
- Para pago total: "Pago total Factura Compra #X"
- Las notas contables de compra se vinculan a la entidad CompraEntity (no a Factura como en ventas)

---

## 📡 Integración con CompraService

La lógica debe implementarse en `CompraServiceImpl.alCrear()` o método similar:
```java
// Después de guardar la compra
BigDecimal totalPagado = suma de pagos registrados;
BigDecimal saldoPendiente = totalCompra - totalPagado;

if (totalPagado > 0) {
    if (saldoPendiente > 0) {
        // Pago parcial
        notaContableService.generarNotaDebitoPagoCompra(
            compra.getId(), 
            totalPagado, 
            usuarioId, 
            metodoPagoPrincipal,
            String.format("Pago parcial Fact: %s - Usuario: %s", 
                numFactura, nombreUsuario)
        );
    } else {
        // Pagado completamente
        notaContableService.generarNotaDebitoPagoCompra(
            compra.getId(),
            totalPagado,
            usuarioId,
            metodoPagoPrincipal,
            String.format("Pago total Factura Compra #%d", compra.getId())
        );
    }
}
```
