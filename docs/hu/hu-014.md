# HU-014: Generación Automática de Cuentas por Cobrar desde Ventas

## Descripción
Cuando se registra una venta con método de pago "CREDITO" o con saldo pendiente (pago parcial), el sistema debe generar automáticamente una cuenta por cobrar vinculada a la venta y al cliente.

## Criterios de Aceptación

### Generación Automática
- [x] Al crear una venta con método de pago "CREDITO", se crea automáticamente una cuenta por cobrar
- [x] Al crear una venta con pago parcial (saldo pendiente > 0), se crea automáticamente una cuenta por cobrar
- [x] La cuenta por cobrar se vincula a:
  - `ventaId`: ID de la venta origen
  - `clienteId`: ID del cliente de la venta
  - `totalDeuda`: Total de la venta (incluye impuestos)
  - `fechaEmision`: Fecha de emisión de la venta
  - `fechaVencimiento`: Puede especificarse en la venta o usar por defecto 30 días
  - `observaciones`: 
    - "Venta #X - Venta a crédito" (si método de pago es CREDITO)
    - "Venta #X - Pago parcial" (si es pago parcial)
- [x] El estado inicial de la cuenta es "activa"
- [x] El número de cuenta se genera automáticamente: CC-YYYYMMDD-XXXX
- [x] Si la venta no tiene cliente (consumidor final), NO se genera cuenta por cobrar

### Detección de Venta a Crédito
- [x] El sistema detecta método de pago "CREDITO" en la lista de pagos
- [x] Si existe al menos un pago con método "CREDITO", se considera venta a crédito
- [x] El saldo pendiente en venta a crédito = total de la venta - suma de otros pagos

### Modificaciones al DTO de Venta
- [x] Agregar campo opcional `fechaVencimiento` en `CreateVentaDto`
  - Tipo: `LocalDateTime`
  - Obligatorio: No
  - Por defecto: fechaEmision + 30 días

### Endpoint de Venta
- [x] `POST /api/ventas` genera cuenta por cobrar si:
  - Hay un método de pago "CREDITO" en la lista de pagos
  - O hay saldo pendiente (pago parcial)
  - Y la venta tiene cliente asignado

### Casos de Prueba
1. **Venta con pago completo (efectivo/tarjeta)**: No genera cuenta por cobrar
2. **Venta a crédito con cliente**: Genera cuenta por cobrar con totalDeuda = total venta
3. **Venta a crédito sin cliente**: No genera cuenta por cobrar (consumidor final)
4. **Venta con pago parcial y cliente**: Genera cuenta por cobrar con saldo pendiente
5. **Venta con pago parcial sin cliente**: No genera cuenta por cobrar
6. **Venta con fechaVencimiento específica**: Usa esa fecha en la cuenta
7. **Venta con múltiples pagos (efectivo + crédito)**: Genera cuenta por cobrar por el saldo

## Implementación

### VentaServiceImpl
- Inyectado `CuentaCobrarService`
- Detección de crédito:
```java
// Detectar si es venta a crédito
boolean esCredito = false;
for (CreateVentaPagoDto pago : dto.getPagos()) {
    if ("CREDITO".equalsIgnoreCase(pago.getMetodoPago())) {
        esCredito = true;
        saldoPendiente = totalFinal.subtract(totalPagado);
        break;
    }
}
```

- Creación de cuenta por cobrar:
```java
// 8. Crear cuenta por cobrar si es crédito o pago parcial (HU-014)
if ((esCredito || esPagoParcial) && dto.getClienteId() != null) {
    CreateCuentaCobrarDto cuentaCobrarDto = new CreateCuentaCobrarDto();
    cuentaCobrarDto.setClienteId(dto.getClienteId());
    cuentaCobrarDto.setVentaId(venta.getId());
    cuentaCobrarDto.setTotalDeuda(totalFinal);
    cuentaCobrarDto.setFechaEmision(venta.getFechaEmision());
    cuentaCobrarDto.setFechaVencimiento(dto.getFechaVencimiento() != null ? 
        dto.getFechaVencimiento() : venta.getFechaEmision().plusDays(30));
    cuentaCobrarDto.setObservaciones(esCredito ? 
        "Venta #" + venta.getId() + " - Venta a crédito" : 
        "Venta #" + venta.getId() + " - Pago parcial");

    cuentaCobrarService.crear(cuentaCobrarDto, empresaId, usuarioId);
}
```
