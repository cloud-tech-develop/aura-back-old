# HU-030: Permisos de Submódulos por Empresa

**Epic:** EP-006 - Sistema de Permisos de Módulos

## Descripción

Como super-admin de la plataforma, quiero activar o desactivar submódulos específicos para cada empresa, controlando el acceso granular a funcionalidades.

## Criterios de Aceptación

- [x] El super-admin puede ver los submódulos activos de una empresa
- [x] El super-admin puede activar un submódulo para una empresa
- [x] El super-admin puede desactivar un submódulo para una empresa
- [x] Al crear una empresa, por defecto todos los submódulos están activos
- [x] Un submódulo solo puede estar activo si su módulo padre también está activo
- [x] La empresa no puede acceder a un submódulo desactivado

## API Endpoints

Incluidos en HU-029:
| Método | Endpoint | Descripción |
|--------|----------|-------------|
| GET | `/api/platform/empresas/{id}/permisos` | Ver permisos de módulos y submódulos |
| PUT | `/api/platform/empresas/{id}/permisos` | Actualizar permisos |

## Estructura del Request

```json
{
  "modulos": [
    {
      "moduloId": 1,
      "activo": true,
      "submodulos": [
        {
          "submoduloId": 1,
          "activo": true
        }
      ]
    }
  ]
}
```

## Reglas de Negocio

1. No se puede activar un submódulo si su módulo padre está desactivado
2. Al desactivar un módulo padre, los submódulos hijos se desactivan automáticamente
3. Solo el super-admin (PLATFORM_ADMIN) puede modificar permisos

## Database Modifications

La tabla `permiso_empresa` ya está definida en HU-029.

### Consideraciones de Implementación

Al actualizar permisos de submódulos:
```sql
-- Verificar que el módulo padre esté activo
SELECT pe.activo 
FROM permiso_empresa pe
WHERE pe.empresa_id = :empresaId 
  AND pe.modulo_id = :moduloId 
  AND pe.submodulo_id IS NULL;

-- Actualizar permiso de submódulo
UPDATE permiso_empresa 
SET activo = :activo, updated_at = CURRENT_TIMESTAMP
WHERE empresa_id = :empresaId 
  AND submodulo_id = :submoduloId;
```
