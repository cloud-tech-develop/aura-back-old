## 👤 Historia de Usuario

**Como** sistema de punto de venta  
**Quiero** generar notas contables de tipo crédito y débito asociadas a facturas  
**Para** registrar ajustes, devoluciones, vueltos y pagos adicionales en el sistema contable de la empresa

---

## 🧠 Descripción Funcional

El sistema debe permitir crear notas contables vinculadas a facturas. Las notas contables pueden ser de dos tipos:
- **CRÉDITO (tipo = 1)**: Representa devoluciones, descuentos adicionales, o sobrantes pagados al cliente
- **DÉBITO (tipo = 2)**: Representa cargos adicionales, intereses, o pagos registrados

Las notas contables deben mantener trazabilidad completa con información del usuario que las registra, método de pago utilizado, y una descripción/nota explicativa.

---

## ✅ Criterios de Aceptación

### Escenario 1: Generación automática de nota crédito por vuelto
- Dado que se registra un pago de $120.000 en una venta de $100.000
- Cuando se completa el proceso de pago
- Entonces:
  - Debe generarse una NotaContable con tipo CRÉDITO
  - El valor debe ser $20.000 (vuelto)
  - Debe estar vinculada a la factura correspondiente
  - La nota debe indicar "Vuelto para el cliente"

### Escenario 2: Generación automática de nota débito por pago adicional
- Dado que existe una venta con pago parcial ($50.000 de $100.000)
- Cuando se registra un nuevo pago de $50.000 completando el total
- Entonces:
  - Debe generarse una NotaContable con tipo DÉBITO
  - El valor debe ser $50.000
  - Debe estar vinculada a la factura correspondiente

### Escenario 3: Crear nota contable manualmente
- Dado que existe una factura registrada
- Cuando un usuario crea una nota contable manualmente
- Entonces:
  - Se debe guardar con todos los campos obligatorios
  - El campo tipo debe ser 1 (CRÉDITO) o 2 (DÉBITO)
  - Debe registrar el usuario que la crea

### Escenario 4: Consulta de notas contables por factura
- Dado que existe una factura con notas contables
- Cuando se consultan las notas de esa factura
- Entonces:
  - Se deben返回 todas las notas vinculadas
  - El orden debe ser por fecha de creación descendente

---

## ❌ Casos de Error

- Si el valor de la nota es menor o igual a 0, debe retornar error 400
- Si la factura no existe, debe retornar error 404
- Si el tipo no es 1 o 2, debe retornar error 400
- Si no hay usuario_id, debe retornar error 400
- Si el método de pago no es válido, debe retornar error 400

## 🗄️ Schema de Base de Datos (PostgreSQL)

### Tabla: nota_contable

```sql
CREATE TABLE nota_contable (
    id BIGSERIAL PRIMARY KEY,
    factura_id BIGINT NOT NULL REFERENCES factura(id),
    valor DECIMAL(12,2) NOT NULL,
    banco VARCHAR(300),
    tipo INTEGER NOT NULL CHECK (tipo IN (1, 2)),
    nota VARCHAR(500) NOT NULL,
    usuario_id INTEGER NOT NULL REFERENCES usuario(id),
    metodo_pago VARCHAR(50) NOT NULL,
    created_at TIMESTAMP NOT NULL DEFAULT NOW(),
    
    CONSTRAINT nota_contable_factura_fk FOREIGN KEY (factura_id) REFERENCES factura(id) ON DELETE CASCADE,
    CONSTRAINT nota_contable_valor_positivo CHECK (valor > 0)
);

CREATE INDEX idx_nota_contable_factura ON nota_contable(factura_id);
CREATE INDEX idx_nota_contable_tipo ON nota_contable(tipo);
CREATE INDEX idx_nota_contable_created_at ON nota_contable(created_at);
```
