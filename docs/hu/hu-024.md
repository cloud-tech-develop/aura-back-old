## 👤 Historia de Usuario

**Como** usuario del sistema  
**Quiero** programar visitas a locales  
**Para** planificar y registrar las visitas de los vendedores a los clientes

---

## 🧠 Descripción Funcional

El sistema debe permitir programar visitas indicando el local, vendedor, fecha y hora programadas. Las visitas pueden ser generadas automáticamente desde las rutas o creadas manualmente. Cada visita tiene un estado (programada, completada, cancelada).

---

## ✅ Criterios de Aceptación

### Escenario 1: Crear visita manualmente
- Dado que el usuario está en la pantalla de programación de visitas
- Cuando hace clic en "Nueva Visita" y selecciona local, vendedor, fecha y hora
-Entonces el sistema crea la visita programada

### Escenario 2: Generar visitas desde ruta
- Dado que el usuario está en los detalles de una ruta
- Cuando hace clic en "Programar Visitas" y selecciona una fecha
-Entonces el sistema genera una visita para cada local de la ruta en el orden establecido

### Escenario 3: Completar visita
- Dado que el usuario está en la lista de visitas
- Cuando selecciona una visita programada y marca como completada
-Entonces el sistema actualiza el estado a "completada" y registra la fecha/hora real

### Escenario 4: Cancelar visita
- Dado que el usuario está en la lista de visitas
- Cuando selecciona una visita y confirma la cancelación
-Entonces el sistema actualiza el estado a "cancelada"

### Escenario 5: Listar visitas
- Dado que el usuario accede a la gestión de visitas
- Cuando la pantalla carga
-Entonces el sistema muestra una lista paginada de visitas de la empresa

### Escenario 6: Filtrar visitas por vendedor
- Dado que el usuario está en la lista de visitas
- Cuando selecciona un vendedor del filtro
-Entonces el sistema muestra solo las visitas de ese vendedor

### Escenario 7: Filtrar visitas por fecha
- Dado que el usuario está en la lista de visitas
- Cuando selecciona un rango de fechas
-Entonces el sistema muestra solo las visitas dentro de ese rango

### Escenario 8: Filtrar visitas por estado
- Dado que el usuario está en la lista de visitas
- Cuando selecciona un estado (programada, completada, cancelada)
-Entonces el sistema muestra solo las visitas de ese estado

### Escenario 9: Programar rutas en calendario
- Dado que el usuario está en la vista de calendario
- Cuando arrastra una ruta y la suelta en una fecha específica
-Entonces el sistema genera las visitas programadas para esa fecha

### Escenario 10: Vista de calendario de rutas
- Dado que el usuario accede a la programación de rutas
- Cuando la pantalla carga
-Entonces el sistema muestra un calendario con las rutas programadas por fecha

### Escenario 11: Confirmar visita en el local
- Dado que el vendedor está en el local de la ruta del día
- Cuando confirma su llegada en la aplicación (botón "Confirmar Llegada")
-Entonces el sistema registra la visita como completada con la hora real de llegada

### Escenario 12: Verificar ubicación para confirmar visita
- Dado que el vendedor está en el local
- Cuando intenta confirmar llegada
-Entonces el sistema verifica que la ubicación actual esté cerca del local (radio configurable) antes de confirmar

### Escenario 13: Confirmar visita sin ubicación
- Dado que el vendedor no tiene GPS disponible
- Cuando confirma la llegada manualmente
-Entonces el sistema registra la visita como completada sin verificación de ubicación

### Escenario 14: Mostrar ruta del día al vendedor
- Dado que el vendedor accede a su panel
- Cuando carga las visitas del día
-Entonces el sistema muestra la lista de locales a visitar en orden con la hora programada

---

## ❌ Casos de Error

- Si el local no existe, mostrar error
- Si el vendedor no existe, mostrar error
- Si la fecha está en el pasado, mostrar advertencia pero permitir
- Si ya existe una visita para el mismo local y fecha/hora, mostrar error

---

## 🔐 Reglas de Negocio

- Una visita pertenece a un local y un vendedor específicos
- Los estados posibles son: programada, completada, cancelada
- Las visitas se filtran por empresa_id del JWT
- Al completar una visita se registra la hora real de visita
- La programación en calendario permite previsualizar las rutas programadas
- La confirmación de visita requiere verificación de ubicación (geofencing)
- El radio de verificación de ubicación es configurable (por defecto 100 metros)
- El vendedor puede confirmar manualmente si no tiene GPS disponible

---

## 🎨 Consideraciones UI/UX

- Calendario para visualizar las visitas programadas
- Indicador de color por estado (verde: completada, amarillo: programada, rojo: cancelada)
- Modal de confirmación al completar o cancelar
- Filtros combinados (fecha + vendedor + estado)

---

## 📡 Requisitos Técnicos

- Endpoints REST para CRUD de visitas
- GET /api/visitas?fecha=YYYY-MM-DD&vendedorId=1&estado=programada
- POST /api/visitas/from-ruta - Generar visitas desde ruta
- GET /api/visitas/calendario - Obtener visitas programadas por rango de fechas
- POST /api/visitas/{id}/confirmar - Confirmar llegada a visita (con coordenadas)
- GET /api/visitas/hoy - Obtener las visitas del día para el vendedor autenticado
- POST /api/rutas/{id}/programar - Programar ruta en una fecha específica
