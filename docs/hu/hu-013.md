# HU-013: Cálculo de Totales de Cuentas por Cobrar y Pagar

## Descripción
Proporciona endpoints para obtener los cálculos agregados de totales de deuda, total abonado y saldo pendiente para cuentas por cobrar y por pagar. Estos cálculos se utilizan para dashboards y reportes financieros.

**Nota**: Esta HU depende de HU-015 para el cálculo correcto de totales desde abonos.

## Criterios de Aceptación

### Obtener Resumen de Cuentas por Cobrar
- [x] Endpoint: `GET /api/cuentas-cobrar/resumen`
- [x] Request Query Params (todos opcionales):
  - `fechaDesde`: fecha inicial de filtro
  - `fechaHasta`: fecha final de filtro
  - `clienteId`: filtrar por cliente específico
  - `estado`: filtrar por estado (activa, pagada, vencida)
- [x] Response: `ApiResponse<CuentaCobrarResumenDto>`
- [x] Campos del resumen:
  - `totalCuentas`: cantidad de cuentas que cumplen el filtro
  - `totalDeuda`: suma de todos los total_deuda
  - `totalAbonado`: suma calculada desde abonos
  - `saldoPendiente`: suma calculada (totalDeuda - totalAbonado)
  - `cantidadActivas`: cuentas con estado 'activa'
  - `cantidadPagadas`: cuentas con estado 'pagada'
  - `cantidadVencidas`: cuentas con estado 'vencida'
- [x] Query SQL con suma agregada desde abonos:
```sql
SELECT 
    COUNT(*) AS total_cuentas,
    SUM(total_deuda) AS total_deuda,
    COALESCE(SUM((SELECT SUM(monto) FROM abonos_cobrar WHERE cuenta_cobrar_id = cc.id AND deleted_at IS NULL)), 0) AS total_abonado,
    SUM(total_deuda) - COALESCE(SUM((SELECT SUM(monto) FROM abonos_cobrar WHERE cuenta_cobrar_id = cc.id AND deleted_at IS NULL)), 0) AS saldo_pendiente,
    COUNT(*) FILTER (WHERE CASE 
        WHEN total_deuda - COALESCE((SELECT SUM(monto) FROM abonos_cobrar WHERE cuenta_cobrar_id = cc.id AND deleted_at IS NULL), 0) <= 0 THEN 'pagada'
        WHEN fecha_vencimiento < NOW() THEN 'vencida'
        ELSE 'activa'
    END = 'activa') AS cantidad_activas,
    COUNT(*) FILTER (WHERE CASE 
        WHEN total_deuda - COALESCE((SELECT SUM(monto) FROM abonos_cobrar WHERE cuenta_cobrar_id = cc.id AND deleted_at IS NULL), 0) <= 0 THEN 'pagada'
        WHEN fecha_vencimiento < NOW() THEN 'vencida'
        ELSE 'activa'
    END = 'pagada') AS cantidad_pagadas,
    COUNT(*) FILTER (WHERE CASE 
        WHEN total_deuda - COALESCE((SELECT SUM(monto) FROM abonos_cobrar WHERE cuenta_cobrar_id = cc.id AND deleted_at IS NULL), 0) <= 0 THEN 'pagada'
        WHEN fecha_vencimiento < NOW() THEN 'vencida'
        ELSE 'activa'
    END = 'vencida') AS cantidad_vencidas
FROM cuentas_cobrar cc
WHERE empresa_id = :empresaId
  AND deleted_at IS NULL
  -- filtros adicionales
```

### Obtener Resumen de Cuentas por Pagar
- [x] Endpoint: `GET /api/cuentas-pagar/resumen`
- [x] Request Query Params (todos opcionales):
  - `fechaDesde`: fecha inicial de filtro
  - `fechaHasta`: fecha final de filtro
  - `proveedorId`: filtrar por proveedor específico
  - `estado`: filtrar por estado (activa, pagada, vencida)
- [x] Response: `ApiResponse<CuentaPagarResumenDto>`
- [x] Misma estructura que cuentas por cobrar (usando abonos_pagar)

### Obtener Totales por Cliente (Cuentas por Cobrar)
- [x] Endpoint: `GET /api/cuentas-cobrar/resumen/cliente/{clienteId}`
- [x] Response: `ApiResponse<ClienteCuentaResumenDto>`
- [x] Campos:
  - `clienteId`, `clienteNombre`, `numeroCuentas`
  - `totalDeuda`, `totalAbonado`, `saldoPendiente`

### Obtener Totales por Proveedor (Cuentas por Pagar)
- [x] Endpoint: `GET /api/cuentas-pagar/resumen/proveedor/{proveedorId}`
- [x] Response: `ApiResponse<ProveedorCuentaResumenDto>`
- [x] Campos:
  - `proveedorId`, `proveedorNombre`, `numeroCuentas`
  - `totalDeuda`, `totalAbonado`, `saldoPendiente`

### Obtener Cuentas Vencidas
- [x] Endpoint: `GET /api/cuentas-cobrar/vencidas`
- [x] Filtra cuentas donde fecha_vencimiento < now() AND saldo_pendiente > 0
- [x] Response: `ApiResponse<List<CuentaCobrarTableDto>>`

- [x] Endpoint: `GET /api/cuentas-pagar/vencidas`
- [x] Filtra cuentas donde fecha_vencimiento < now() AND saldo_pendiente > 0
- [x] Response: `ApiResponse<List<CuentaPagarTableDto>>`

## Seguridad
- [x] Requiere JWT válido
- [x] Filtra por empresa_id del token
- [x] Roles permitidos: ADMIN, CONTADOR, GERENTE

## Notas Técnicas
- Los cálculos se realizan directamente en la base de datos (SUM) para mejor rendimiento
- El campo saldo_pendiente se calcula como: total_deuda - SUM(abonos)
- Las cuentas con deleted_at no se incluyen en los cálculos
- El endpoint de vencidas recalcula el estado dinámicamente
