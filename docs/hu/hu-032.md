# HU-032: Endpoint Público de Permisos

**Epic:** EP-006 - Sistema de Permisos de Módulos

## Descripción

Como frontend, quiero obtener los módulos y submódulos activos para una empresa sin necesidad de autenticación completa, para mostrar u ocultar opciones del menú.

## Criterios de Aceptación

- [x] Endpoint accesible públicamente (sin auth)
- [x] Recibe el NIT de la empresa como parámetro
- [x] Retorna solo los módulos y submódulos activos
- [x] El response incluye código y nombre para construir menús dinámicos

## API Endpoint

| Método | Endpoint | Descripción |
|--------|----------|-------------|
| GET | `/api/public/empresas/{nit}/permisos` | Obtener permisos públicos |

## Response

```json
{
  "empresaId": 1,
  "nit": "900123456",
  "modulos": [
    {
      "codigo": "catalogo",
      "nombre": "Catálogo",
      "activo": true,
      "submodulos": [
        {
          "codigo": "productos",
          "nombre": "Productos",
          "activo": true
        }
      ]
    }
  ]
}
```

## Reglas de Negocio

1. Solo retorna módulos y submódulos con activo = true
2. Si la empresa no existe, retorna 404
3. El endpoint es público (no requiere JWT)
4. No retorna información sensible

## Implementación

### Query SQL

```sql
SELECT 
    m.id as modulo_id,
    m.codigo as modulo_codigo,
    m.nombre as modulo_nombre,
    s.id as submodulo_id,
    s.codigo as submodulo_codigo,
    s.nombre as submodulo_nombre
FROM empresa e
JOIN permiso_empresa pe ON pe.empresa_id = e.id
JOIN modulo m ON m.id = pe.modulo_id AND pe.submodulo_id IS NULL
LEFT JOIN permiso_empresa pes ON pes.empresa_id = e.id AND pes.modulo_id = m.id
LEFT JOIN submodulo s ON s.id = pes.submodulo_id
WHERE e.nit = :nit 
  AND pe.activo = true 
  AND (s.id IS NULL OR s.activo = true)
ORDER BY m.orden, s.orden;
```

### Response DTO

```java
public record PermisoPublicoDto(
    Long empresaId,
    String nit,
    List<ModuloPermisoDto> modulos

public record ModuloPermisoDto(

```
