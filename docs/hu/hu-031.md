# HU-031: Validación de Permisos en Controladores

**Epic:** EP-006 - Sistema de Permisos de Módulos

## Descripción

Como sistema, quiero verificar los permisos de módulos y submódulos en cada solicitud HTTP para garantizar que las empresas solo accedan a funcionalidades autorizadas.

## Criterios de Aceptación

- [x] Cada controlador verifica el permiso del módulo antes de ejecutar
- [x] Si la empresa no tiene permiso, retorna error 403 Forbidden
- [x] Los controladores existentes verifican permisos automáticamente
- [x] La verificación usa el empresaId del token JWT

## Implementación

### Anotación Personalizada

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface RequerirPermiso {
    String modulo();
    String submodulo() default "";
}
```

### Ejemplo de Uso

```java
@RestController
@RequestMapping("/api/productos")
public class ProductoController {
    
    @GetMapping
    @RequerirPermiso(modulo = "catalogo", submodulo = "productos")
    public ResponseEntity<...> listar() { ... }
}
```

### Filter/Interceptor

Alternativamente, usar un interceptor que verifique permisos basados en la ruta del endpoint.

## Reglas de Negocio

1. La verificación se hace después de autenticación (ya hay empresaId en token)
2. Los endpoints públicos no requieren verificación
3. El super-admin (PLATFORM_ADMIN) tiene acceso a todo

### Configuración

```java
@Configuration
@RequiredArgsConstructor
public class WebConfig implements WebMvcConfigurer {
    
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(permisoInterceptor())
            .addPathPatterns("/api/**")
            .excludePathPatterns(
                "/api/auth/**",
                "/api/public/**",
                "/api/platform/**"
            );
    }
}
```

### Rutas sin Permiso (siempre accesibles)

- `/api/auth/**` - Autenticación
- `/api/public/**` - Endpoints públicos
- `/api/platform/**` - Administración de plataforma (solo PLATFORM_ADMIN)
